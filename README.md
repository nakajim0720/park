# ボウリング受付管理システム (Web版)

## 概要

本システムは、ボウリング場における受付業務をデジタル化し、効率的かつ正確なオペレーションを実現するためのローカルWebアプリケーションです。
お客様の予約QRコードを読み取り、安全に情報を復号した上で、レーンの割り当て、待機リストの管理、履歴の保存までをワンストップで行います。

## 主な機能

* **QRコード連携**: `html5-qrcode`を利用した、ブラウザ上での高速なQRコード読み取り。
* **セキュアなデータ復号**: 予約データはRSA公開鍵で暗号化されてGAS (Google App Script)から取得され、ローカルの秘密鍵(`private.pem`)を使って安全に復号されます。
* **高機能な受付コンソール**:
  * プレイヤー人数の動的追加・削除（リアルタイムな人数バッジ表示付き）。
  * レーンの入力補助（「〜」や「＆」を用いた複数レーン入力）と、無効な数字（0以下、37以上）の自動ブロック。
  * プラン（備考）の選択必須化による案内漏れ防止。
  * 案内完了前の「最終確認モーダル」によるヒューマンエラー防止。
* **待機中リスト管理**: レーン空き待ちのお客様情報を一時保存。状況に応じたデータの呼び出し、更新、削除が可能です。
* **案内履歴のローカル保存とスマート検索**:
  * 案内完了時、自動的に `csv_data` ディレクトリ内に日別のCSV履歴を保存。
  * 「1-4」や「1,3」といった複雑なレーン表記もシステムが自動解釈し、当日案内したお客様の履歴を瞬時に検索・ポップアップ表示します。

## ディレクトリ構成

```text
reception_project/
 ├── private.pem           # RSA秘密鍵（絶対に外部ネットワークに公開しないこと）
 ├── app.py                # バックエンドサーバー本体 (Flask)
 ├── requirements.txt      # 必要なPythonライブラリ一覧
 ├── csv_data/             # 受付履歴CSVの自動保存先ディレクトリ
 └── templates/
      └── index.html       # フロントエンドUI (HTML/CSS/JavaScript)
```
