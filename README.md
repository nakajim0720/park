# ボウリング受付管理システム (Web版)

## 概要

本システムは、ボウリング場におけるフロント受付業務を完全デジタル化し、効率的かつセキュアなオペレーションを実現するためのローカルWebアプリケーションです。
お客様ご自身のスマートフォンからの事前入力、または店舗設置のキオスク端末からの自動発券に対応。発行されたQRコードをスタッフ用コンソールで読み取ることで、レーン割り当て、待機リスト管理、人数変更などの操作履歴（監査ログ）の保存までをワンストップで行います。

## 主な機能と特徴

### 1. ユーザーインターフェース (UI)

* **お客様用スマホフォーム (`index.html`)**
  * 来店前にスマホで入力し、受付用QRコードを発行。
  * スタート画面での「事前予約ではない」旨の注意喚起により、フロントでのクレームを防止。
  * 電話番号の自動整形（全角→半角、数字以外の文字除去）。
* **店舗キオスク端末 (`store_input.html`)**
  * スマホをお持ちでないお客様向けの店舗設置用入力フォーム。
  * 80mmレシートプリンターに最適化。ブラウザのキオスク印刷設定（`--kiosk-printing`）と連携し、ダイアログなしで受付票を全自動発券。発券後は自動で初期画面にリセットされます。
* **スタッフ用受付コンソール (`templates/index.html`)**
  * カメラや専用スキャナを用いたQRコードの高速読み取り。
  * 「案内済データ」の復元機能（例：読取時に「1-2レーンに案内済」と表示され、途中合流のプレイヤー追加などが可能）。
  * 待機リストへの一時保存と呼び出し機能。
  * 複雑なレーン検索（「1-4」や「1,3」など）に自動対応。

### 2. イミュータブル（追記型）データモデル

* 案内済みデータの「人数」や「プラン」を後から変更（再編集）した場合、過去のデータを上書きして消去するのではなく、**常に新しい行としてCSVの末尾に追記**します。
* 変更されたデータには自動的に「編集済」フラグが付与されるため、「誰がいつデータを変更したか」という操作の証拠（監査ログ）が完全に残ります。

### 3. ゼロトラスト・セキュリティ設計

* **Web Crypto APIによる完全ローカル暗号化**: 外部ライブラリを使用せず、ブラウザ標準のセキュアなAPIで個人情報（氏名・連絡先・法人名）をRSA-OAEP暗号化。
* クラウド（Googleフォーム/GAS）には暗号化された状態でのみ送信・保存され、店舗PC内の秘密鍵（`private.pem`）でのみ復号される仕組みのため、万が一クラウドが不正アクセスされても個人情報は漏洩しません。

## ディレクトリ構成

```text
reception_project/
 ├── private.pem           # RSA秘密鍵（※絶対に外部に流出させないこと）
 ├── web.py                # バックエンドサーバー本体 (Flask)
 ├── requirements.txt      # 必要なPythonライブラリ一覧
 ├── csv_data/             # 受付履歴CSVの保存先（日別で自動生成）
 ├── index.html            # お客様用スマホ受付フォーム (Webサーバー等で公開)
 ├── store_input.html      # 店舗設置用・自動発券キオスクフォーム
 ├── style.css             # 各種フォームの共通デザインファイル
 └── templates/
      └── index.html       # スタッフ用 受付コンソール画面
```
